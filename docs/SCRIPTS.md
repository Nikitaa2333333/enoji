# 🛠️ Скрипты и Массовые Изменения (SCRIPTS.md)

Этот файл — памятка и лог для всех автоматизаций в проекте Туроператора. Его цель — гарантировать, что каждый новый скрипт не «сломает» работу предыдущих и сохранит дизайн-стандарты «Pro Max».

---

## 🚦 Правила Безопасности

1. **Идемпотентность** — скрипт должен быть написан так, чтобы его можно было запустить 10 раз подряд без изменения результата (проверка на наличие изменений перед их внесением).
2. **Атомный тест** — сначала запускаем скрипт только на ОДНОМ файле (например, `memo-egypt.html`), проверяем глазами, и только потом натравливаем на все файлы.
3. **BS4 вместо Regex** — по возможности использовать библиотеку `BeautifulSoup4` для манипуляции HTML. Regex допустим только для простых замен в текстовых блоках.
4. **Маркеры (Anchors)** — использовать HTML-комментарии типа `<!-- ═══ CONTENT AREA ═══ -->` как якоря для скриптов.
5. **No Style Overwrite** — скрипты не должны удалять классы `nav-link`, `object-cover`, `rounded-[2.5rem]` и другие элементы дизайн-системы.

## 🏗️ Стандартные паттерны кода

### Как искать заголовки для навигации
```python
# Правильный паттерн (ищем только в контенте, игнорируем футер/хедер)
content_area = soup.find('div', id='memo-content-area')
headings = content_area.find_all('h2') if content_area else []
```

### Как заменять блоки
```python
# Добавляем ID если его нет для якоря
import slugify
for h2 in headings:
    if not h2.get('id'):
        h2['id'] = generate_slug(h2.text)
```

## 🧠 Уроки и Исправленные Ошибки (Lessons Learned)

| Ошибка | Причина | Решение (The Fix) |
|---|---|---|
| **404 на Vercel** | Vite не видел HTML файлы, которые не были `.index.html` | Настроили `vercel.json` для прямой отдачи статики, минуя ненужный билд. |
| **Сломанный скролл** | ID заголовков менялись скриптами, а ссылки в меню оставались старыми | Теперь `rebuild_nav.py` всегда пересобирает меню ПОСЛЕ изменения заголовков. |
| **Пустые рамки фото** | Оставались блоки `tild...` после удаления битых ссылок | Скрипт `cleanup_memo.py` теперь удаляет весь родительский контейнер, а не только `<img>`. |
| **CAPS LOCK в H1/H2** | Прямой экспорт из Тильды | Правило в `gemini.md` №9: Всегда переводить в нормальный регистр программно. |
