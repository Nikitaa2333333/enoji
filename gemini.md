## 📋 Правила работы с текстом

### ❌ Проблема: ИИ искажает текст
- Сокращает или перефразирует
- «Улучшает» стиль
- Теряет часть предложений

### ✅ Решение: давать текст через файл

**Способ 1 — файл .txt (РЕКОМЕНДУЕТСЯ)**
Создай файл `[страна]_content.txt` по формату:
```
СТРАНА: Египет
SLUG: egypt
HERO_PHOTO: egypt_hero.jpg

ТЕКСТ:
Первый абзац...

Второй абзац...

ФОТО: egypt_photo.jpg

Третий абзац...

РЕГИОН: Хургада
ФОТО: hurghada.jpg
Текст о Хургаде...
```
Скажи: «возьми текст из файла egypt_content.txt и вставь дословно»

**Способ 2 — тройные кавычки в чате**
Оберни текст в ` ```VERBATIM ``` ` — сигнал «не трогать»

## ⚖️ Правила поведения

1. **Russian Language Only** — все ответы только на русском.
2. **No Auto-Run** — все команды передаются пользователю для ручного запуска.
3. **Template First** — новую страницу всегда копировать из `template.html`.
4. **Текст дословно** — никогда не перефразировать и не сокращать текст пользователя.
5. **Параграфы раздельно** — каждый смысловой абзац = отдельный `<p>`.
6. **Стиль не трогать** — CSS-классы не меняются при правках контента.
7. **PAGE_COUNTRY** — в каждой странице заменять эту переменную на название страны.
9. **NO CAPS** — использование Caps Lock запрещено в заголовках `<h1>`, `<h2>` и основном тексте. Все тексты из исходников Tilda должны быть переведены в нормальный регистр.
10. **NO PHOTO PLACEHOLDERS** — если в тексте встречается строка типа «Фото: tild...», она должна быть удалена вместе с пустым контейнером. Мусорный код о ненайденных фото не выставлять.
11. **MEMO PAGES** — на страницах памяток («Памятка туристу») правила о капсе и фото-плейсхолдерах соблюдать особенно строго.
12. **AUTOMATION LOG** — любое массовое изменение через скрипт ДОЛЖНО фиксироваться в `SCRIPTS.md` с описанием «до/после».
